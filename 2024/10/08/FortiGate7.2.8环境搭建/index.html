前言

环境搭建

FortiGate中没有类似Linux操作系统中的shell，需要手动提取固件系统并向其中添加，环境搭建基本步骤：

  1. 获取文件系统rootfs.gz
  2. 解密rootfs.gz（7.2.8以上版本需要）
  3. 解压文件系统并添加后门和busybox
  4. 重打包rootfs.gz
  5. 内核调试绕过对文件系统的完整性检查

Tips：
FortiGate automatic repack script项目是步骤2、3和5的自动化脚本，可以结合使用解放双手busybox下载地址：https://www.busybox.net/
如果需要调试，可下载静态编译的gdb，在重打包时添加

固件提取

在低版本中rootfs.gz文件在系统初始化后的disk1.vmdk磁盘文件中，只需要需要将其挂载到一个linux虚拟机上来提取即可

在7.2.8版本中，rootfs.gz是经过加密的。对此网上有不少详细的分析文章，也提供了相应的解密工具：
optistream/fortigate-crypto: This repo provides tools for FortiGate firmware rootfs decryption (github.com)
rrrrrrri/fgt-gadgets: Fortigate related tools (github.com)
但以上方法只在7.4.x系列版本中测试过，实测发现7.2.8以上的版本并不适用，以下是我的提取方法：
无论使用哪种加密方法，在系统启动时都会执行解密逻辑进行解密，定位到解密逻辑的位置后，只需通过动态调试，将解密后的压缩包从内存中dump下来即可获取解密的压缩包
在7.2.8版本中，解密逻辑在fgt_verify_decrypt函数中：

chacha20_docrypt函数对压缩包进行了解密参数分别为：

  • v7是密钥
  • v0是密文，也就是压缩包的内容
  • v1是密文长度
    此时将断点设在该函数执行完成处，动态调试使用gdb命令dump memory filename start start+len即可dump内存到文件，此处filename是保存的文件名，start是解密后的压缩包在内存中的起始地址，len是压缩包长度

    如何进行调试可参考[内核调试](### 内核调试)部分

完整性检查绕过

FortiGate在每次启动时都会对文件系统的完整性进行校验，上面步骤修改了文件系统因此需要手动绕过这些检查
需要绕过的有以下几个点：

  • 替换/sbin/init字符串为/bin/init
  • 跳过do_halt函数
  • 绕过签名检查和文件系统解密

首先从前面挂载的磁盘中复制出内核文件flatkc和init文件，这两个文件中包含了对文件系统的完整性校验

内核文件flatkc是一个Image类型的文件，需要通过这个项目将其转换为ELF格式文件后才能反编译

使用IDA打开init（文件比较大，反编译的时间会比较久），然后通过字符串do_halt来定位do_halt函数
定位到该函数后修改其开头的指令为xor rax, rax ; ret

使其能被直接跳过而不是执行关机操作
需要跳过该函数是因为该函数在main函数中对文件系统检查失败时会被调用来关闭系统

修改方法这里就不详细说明了，需要说明的是使用自动化脚本重打包时只需要将修改后的文件命名为init.patched并复制到项目目录即可

下一步是绕过内核文件flatkc中的检查，因为内核文件不是一个ELF文件，而将修改后的ELF文件转为Image类型文件也是个很麻烦的操作，所以这里通过内核调试的方法来进行修改和绕过
需要的准备工作就是逆向flatkc文件并记录需要修改的地址

首先定位字符串/sbin/init/，记录该字符串地址在后续内核调试时将该字符串修改为/bin/init，在7.2.8版本中该字符串地址为0xFFFFFFFF813D0E8B

/sbin/init程序会执行执行我们重打包时的工作然后调用/bin/init，不需要再执行一遍所以直接替换成/bin/init

接着定位函数fgt_verify_initrd，这个函数会提取rootfs.gz的最后0x100 byte作为签名然后进行验证，验证通过后会调用前面提到的fgt_verify_decrypt函数对rootfs.gz进行解密
记录fgt_verify_initrd函数的地址0xFFFFFFFF8170B488，在后续内核调试时修改其直接返回0来绕过，这样既能绕过签名检查，还不需要执行rootfs.gz的解密，使得我们无需将重打包的文件系统再次进行加密

还有一处需要绕过的是fos_process_appraise_constprop_0函数，该函数用于对文件进行校验，同样的将其修改为返回0即可，函数地址为0xFFFFFFFF8053FA0B
